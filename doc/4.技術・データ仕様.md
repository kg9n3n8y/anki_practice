# 技術・データ仕様

## ビルド・デプロイ

- **ホスティング**：GitHub Pages（静的ファイルのみ）
- **公開 URL**：`https://kg9n3n8y.github.io/anki_practice/`
- **base path**：`/anki_practice/`（環境変数 `VITE_BASE_PATH`。GitHub Actions では `/${リポジトリ名}/` を自動設定）
- **スタック**：Vite + React + TypeScript + React Router + vite-plugin-pwa
- **SPA ルーティング**：ビルド時に `dist/404.html` を `index.html` から生成

## OGP・Twitterカード

- `index.html` に Open Graph / Twitter Card メタタグを設定
- 共有画像: `public/thumbnail.png`（公開 URL: `https://kg9n3n8y.github.io/anki_practice/thumbnail.png`）
- カード種別: `summary_large_image`
- デプロイ後に [X Card Validator](https://cards-dev.twitter.com/validator) や Facebook シェアデバッガーで表示確認するとよい（キャッシュの反映に時間がかかることがある）

## PWA

- Service Worker（`vite-plugin-pwa` / Workbox）で **アプリ本体と取り札画像（`public/torifuda` 全201枚）をプリキャッシュ**。初回オンライン訪問後はオフラインでも SPA ルーティング・練習・画像表示が可能
- 起動時に `warmTorifudaCache()` で `torifuda-images` キャッシュを補完（base path 付き URL。GitHub Pages の `/anki_practice/torifuda/` にも対応）
- オフライン時の画面遷移: `navigateFallback` → `index.html`
- トップ初回表示時に「ホーム画面に追加」案内（`PwaInstallHint`）。キー `pwaInstallHintDismissed`。standalone 表示中は出さない
- manifest（`vite.config.ts`）
    - `name`：**かるた暗記練**
    - `short_name`：**暗記練**
- アイコン：`public/pwa-icon-180.png`（iOS）、`pwa-icon-192.png`、`pwa-icon-512.png`（manifest / maskable）

## 定位置 JSON スキーマ（v2・保存形式）

```json
{
  "version": 2,
  "updatedAt": "2026-06-04T12:00:00.000Z",
  "areas": {
    "leftUpper": [3, 12],
    "rightUpper": [],
    "leftMiddle": [7],
    "rightMiddle": [42],
    "leftLower": [],
    "rightLower": [88, 91]
  }
}
```

- 各 `areas` の配列は、そのエリア内の歌番号を**並び順どおり**に列挙（左エリアは左→右＝外側→内側、右エリアは右揃え表示で外側→内側）
- `no`：1〜100。重複はインポート時に除外（先勝ち）

### コンパクトテキスト形式（共有・コピー用）

```
# position v2
leftUpper:3,12
rightMiddle:42,88
```

- 1行1エリア。歌番号はカンマ区切り。空エリアは行を省略
- 定位置編集画面の「テキストをコピー」でクリップボードへ出力
- トップの取りこみはコンパクトテキストのみ（`parsePositionText`）。JSON 貼り付けは非対応
- 旧ヘッダ `# teigi v2` もパース時に無視して読み込み可能

### v1 からの移行

```json
{
  "version": 1,
  "updatedAt": "...",
  "placements": [{ "no": 1, "area": "leftUpper" }]
}
```

- インポート・読み込み時に v2 へ変換。同一エリア内の順序は `fudalist.order` 昇順で初期化

## 成績1件のデータ

```json
{
  "at": "2026-06-04T12:00:00.000Z",
  "mode": "both",
  "cardCount": 50,
  "memorizeMinutes": 15,
  "questionCount": 100,
  "correctCount": 85,
  "confirmSeconds": 320
}
```

- `questionCount`：出題リストの決まり字の数（正答率の分母）
- `correctCount`：正答の合計（近接正解は 0.5 ずつ加算。小数になりうる）
- `confirmSeconds`：確認モード全体の所要秒数

## 効果音

- 専用音声ファイルは使わない
- 確認モードの正誤時に Web Audio API（`src/lib/beep.ts`）で合成音を再生
    - 正解：明るい2音のチャイム
    - 近接正解：中間域の短い2音
    - 不正解：低めの柔らかい2音
- 確認モード開始時に `preloadBoardImages` で表向き画像を先読み
- 正誤判定は `src/lib/confirmAnswer.ts`（`correct` / `near` / `incorrect`）。`near` は正答数に 0.5 加算

## 「両方」モードの枚数配分

- 合計枚数 `N` は 10〜50 の **10枚単位** のため常に偶数
- 自陣・相手陣はそれぞれ **`N / 2` 枚**

## エリア内配置アルゴリズム（簡易）

1. 各陣・各エリアに割り当てる枚数を決める（最大7枚/エリア、合計が設定枚数と一致）
   - その陣の枚数が **6枚以上**（6エリア分）のときは **全6エリアに最低1枚** を保証し、残りをランダムにばらつき配分
   - 5枚以下のときは、ランダムに選んだエリアに1枚ずつ
2. **定位置適用時（自陣）**：`computeAreaTargets` は使わない。登録配列の順に、選出に含まれる歌をそのエリアへ配置（**エリアあたり最大7枚**。8枚目以降の登録分は未配置扱いで空きスロットへランダム配分）。0枚のエリアを許容。残りの選出札も各エリアの空きへランダム配分
3. **ランダム配置時（相手陣・定位置OFF）**：`computeAreaTargets` で枚数を割り振ったあと配置。各エリア内は外側詰め（左列は左端、右列は右端から順に `push`）
4. 左右の枚数バランスは、可能な範囲で各段の左右差が小さくなるよう配分

## 盤面レイアウト

- 1段の幅 = 左7枚 + 隙間1枚 + 右7枚（計15単位、`BOARD_ROW_UNITS`）。`--fuda-w` はコンテナ幅 ÷ 15 × 0.95
- `MAX_CARDS_PER_AREA = 7`（`src/lib/areas.ts`）
- 暗記メイン画面の shell 最大幅：PC で約 1040px。定位置編集：PC で最大 1180px（`100vw` を超えない）
