# 技術・データ仕様

## ビルド・デプロイ

- **ホスティング**：GitHub Pages（静的ファイルのみ）
- **base path**：リポジトリ名未確定のため、環境変数（例：`VITE_BASE_PATH`）で切り替え可能にする。確定後に設定を固定する。
- **スタック**：Vite + React + TypeScript（推奨。変更時は本ドキュメントを更新する）

## PWA

- Service Worker でアセットをキャッシュしオフライン利用可能にする
- `manifest.json` を配置（`name` / `short_name` はアプリ名確定後に更新可）
- アイコン画像は後から追加。未設定時はデフォルトまたはプレースホルダでビルド可能とする

## LocalStorage キー（案）

| `teigi` | 自陣定位置 |

## 定位置 JSON スキーマ（v1）

```json
{
  "version": 1,
  "updatedAt": "2026-06-03T12:00:00.000Z",
  "placements": [
    { "no": 1, "area": "leftUpper" },
    { "no": 42, "area": "rightLower" }
  ]
}
```

- `no`：`fudalist` の歌番号（1〜100）
- `area`：自陣6エリア ID（実装で以下いずれかの命名を統一する）
    - `leftUpper` / `rightUpper` / `leftMiddle` / `rightMiddle` / `leftLower` / `rightLower`
- 同一 `no` が重複した場合はインポート時にエラーまたは後勝ちで処理（実装でどちらかを明記）

## 成績1件のデータ（案）

```json
{
  "at": "2026-06-03T12:00:00.000Z",
  "mode": "both",
  "cardCount": 50,
  "memorizeMinutes": 15,
  "questionCount": 100,
  "correctCount": 85,
  "confirmSeconds": 320
}
```

- `questionCount`：出題した決まり字の数（正答率の分母）
- `confirmSeconds`：確認モード全体の所要秒数

## 効果音

- 専用音声ファイルは Phase 1 では用意しない
- 正誤時は Web Audio API の短いビープ、または無音でもよい

## 「両方」モードの枚数配分

- 合計枚数 `N` は 10〜50 の **10枚単位** のため常に偶数
- 自陣・相手陣はそれぞれ **`N / 2` 枚**（例：N=50 → 各25、N=30 → 各15）

## エリア内配置アルゴリズム（簡易）

1. 各陣・各エリアに割り当てる枚数を決める（最大7枚/エリア、合計が設定枚数と一致）
   - その陣の枚数が **6枚以上** のときは **全6エリアに最低1枚** を保証し、残りをランダムにばらつき配分
   - 5枚以下のときは、ランダムに選んだエリアに1枚ずつ（全エリアには置けない）
2. 定位置適用時は、該当歌を指定エリアのキューに入れる
3. 各エリア内では、左列は左端から、右列は右端から順に札を詰める（外側詰め）
4. 左右の枚数バランスは、可能な範囲で各段の左右差が小さくなるよう配分（厳密ルールなし）
