# 機能要件

## 定位置・配置

- **定位置編集（自陣のみ）**：決まり字をドラッグ＆ドロップで自陣の6エリアのいずれかに割り当てる。各エリア内の並び順はユーザーが D&D で決めて保存する（未配置プールは `order` 昇順）。
- **定位置の保存/読み込み**：自陣の定位置のみ JSON（v2）でエクスポート（定位置編集画面）・インポート（トップ画面）。v1 JSON は読み込み時に v2 へ自動変換。相手陣・両陣を手動で組む盤面作成画面は不要。
- **ランダム配置生成**：練習開始時に、100首から設定枚数分を完全ランダムで選出し、各陣の6エリアへ配置する。配置ルールは「各エリア最大6枚」「左右均等を意識しつつ外側詰め」の簡易ルールとする（左右偏りの詳細ルールは設けない）。
- **定位置の適用**：定位置 ON かつ自陣に札がある場合、登録済みの歌は指定エリアへ、**保存した並び順**で配置する。今回の選出に含まれない歌の定位置は無視し、その他の自陣札はランダム配置する。相手陣は常にランダム配置。

## 暗記・確認機能

- **暗記モード**：画面上に配置された札を確認し、暗記する。
    - モードは「相手陣のみ」「自陣のみ」「両方」の3種
    - 制限時間は1分〜15分、1分単位で設定（デフォルト15分）
- **確認モード**（暗記後のテスト。独立したテストモード画面は設けない）：
    - 盤上の札をすべて裏向きにし、決まり字を提示して正しい位置（または「なし」）をタップし、即座に正誤判定する
    - **終了条件**：盤上の全札が表向きになった時点で終了
    - **出題数**：盤で使用している枚数の2倍（例：盤30枚 → 60問）
    - **出題札の選定**：盤で使用中の札 ＋ 残りの札からランダムで不足分を選ぶ（重複なし）
    - **出題順**：
        - 決まり字順（デフォルト）：選定した札を `order` の昇順で出題
        - ランダム順：選定した札を完全ランダム順で出題
    - **操作ルール**：
        - 正解して表になった札は、再タップしても反応しない
        - 不正解で表になった札は、再度出題されない
    - **効果音**：専用ファイルは使わず、Web Audio API で短い合成音（正解は明るい2音、不正解は低めの2音）

## データ管理

- **札データ**：全100首（`fudalist.js`、画像は `torifuda/`）
- **成績管理**：
    - 正答率（正答数 / 出題した決まり字の数）
    - 確認モード全体にかかった時間（1問ごとの回答時間は記録しない）
    - 暗記の枚数設定・制限時間などのセッション情報
    - 直近5回分を LocalStorage に保持し、トップ画面に表示

## UI/UX 要件

- 操作性：直感的なインターフェース
- 視認性：実際の畳の配置に近いビジュアル（陣間・札間隔は相対レイアウトで近似）
- 定位置編集画面：決まり字は縦書き表示（必須）。長い決まり字（5・6文字）はコンパクト表記（例：`よのなかよ` → `よの・よ`）
- **モバイル陣ズーム**：Phase 2 以降（Phase 1 では対象外）。詳細は下記

### モバイル陣ズーム（Phase 2）

Phase 1 では、盤面全体が画面幅に合わせて自動縮小表示される（`--fuda-w` をコンテナ幅から算出）。スマホでは取り札が小さくなり、暗記中に札の絵や位置を細かく見づらくなる。

Phase 2 で想定する機能の例：

- **ピンチズーム**や**拡大ボタン**で、自陣・相手陣のいずれか（または全体）を一時的に拡大表示する
- 拡大中は**パン（ドラッグ）**で見たいエリアを移動できる
- 暗記モード向け（確認モードでタップ精度が落ちないよう、操作は分離する想定）

優先度は「スマホで暗記練習するユーザーが多い場合は高め」。PC では現状の自動フィットで十分なことが多い。

## 非機能要件

- **デプロイ先**：GitHub Pages（サーバー・DB なし）
- **公開 URL**：`https://kg9n3n8y.github.io/anki_practice/`（`VITE_BASE_PATH=/anki_practice/`、GitHub Actions で `dist/` をデプロイ）
- **データポータビリティ**：自陣定位置の JSON インポート（トップ）・エクスポート（定位置編集画面のみ）
- **オフライン**：Service Worker + Web App Manifest（表示名「かるた暗記練」。アイコンは後から追加）
- **パフォーマンス**：札の表示・画面遷移がスムーズであること

## 技術スタック

実装時の選定（ドキュメント上の目安）：

- フロントエンド：Vite + React（静的ビルド、GitHub Pages 向け）
- ドラッグ＆ドロップ：dnd-kit（定位置編集画面）
- ストレージ：ブラウザ LocalStorage
- 詳細（JSON スキーマ等）：`4.技術・データ仕様.md` を参照
