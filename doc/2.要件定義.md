# 機能要件

## 定位置・配置

- **定位置編集（自陣のみ）**：決まり字をドラッグ＆ドロップで自陣の6エリアのいずれかに割り当てる。各エリア内の並び順はユーザーが D&D で決めて保存する（未配置プールは `order` 昇順）。
- **定位置の保存/読み込み**：ブラウザ内は v2 JSON（LocalStorage）。共有はコンパクトテキストのコピー＆ペースト（編集画面でコピー、トップで取りこみ）。旧 v1 JSON は LocalStorage 読み込み時に v2 へ自動変換。相手陣・両陣を手動で組む盤面作成画面は不要。
- **ランダム配置生成**：練習開始時に、100首から設定枚数分を完全ランダムで選出し、各陣の6エリアへ配置する。配置ルールは「各エリア最大7枚」「左右均等を意識しつつ外側詰め」の簡易ルールとする（左右偏りの詳細ルールは設けない）。
- **定位置の適用**：定位置 ON かつ自陣に札がある場合、**今回の選出に含まれ、かつ定位置が登録されている歌は必ず**指定エリアへ **保存した並び順**で配置する（エリアが0枚のまま残ってもよい。ランダム配分による「各エリア最低1枚」は行わない）。選出に含まれるが定位置未登録の歌は、空きスロット（各エリア最大7枚）へランダム配置。選出に含まれない歌の定位置は無視。相手陣は常にランダム配置。

## 暗記・確認機能

- **暗記モード**：画面上に配置された札を確認し、暗記する。
    - モードは「相手陣のみ」「自陣のみ」「両方」の3種
    - 制限時間は1分〜15分、1分単位で設定（デフォルト5分）
- **確認モード**（暗記後のテスト。独立したテストモード画面は設けない）：
    - 盤上の札をすべて裏向きにし、決まり字を提示して正しい位置（または「なし」）をタップし、即座に正誤判定する
    - **終了条件**：盤上の全札が表向きになった時点で終了
    - **出題数**：盤で使用中の枚数 ＋ **確認モードの空札の枚数**（スタート画面で設定。デフォルト15枚／両方モード時は30枚）
    - **出題札の選定**：盤で使用中の札 ＋ 盤外の札から「確認モードの空札の枚数」分をランダムで選ぶ（重複なし。盤外候補が足りない場合は可能な範囲まで）
    - **出題順**：
        - 決まり字順（デフォルト）：選定した札を `order` の昇順で出題
        - ランダム順：選定した札を完全ランダム順で出題
    - **操作ルール**：
        - 正解して表になった札は、再タップしても反応しない
        - 不正解・近接正解で正解位置が表になった札は、再度出題されない
        - 出題リストは循環するが、**終了条件は盤上の全札が表向き**（盤外の歌だけが残っていても、盤が終われば結果画面へ）
        - 解答後、次の出題へ進むまで約 0.7 秒の間隔がある
    - **近接正解**：盤上の歌で、正解札と**同一エリア**かつ配列上で**左右隣**（index ± 1）の札をタップしたとき。正答 0.5 加算。決まり字は緑、正解位置の札を表向きにし **△** を重ね表示
    - **正誤の見せ方**：
        - ツールバーの決まり字の色（正解＝赤、不正解＝青、近接正解＝緑）
        - 盤上の歌のみ、表にした札の上に ○（CSS の円）/×/△ を 20% 不透明で表示（確認モード終了まで保持）
    - **効果音**：専用ファイルは使わず、Web Audio API で短い合成音（正解・近接正解・不正解でそれぞれ別パターン）。ON/OFF 設定はない

## データ管理

- **札データ**：全100首（`fudalist.js`、画像は `torifuda/`）
- **成績管理**：
    - 正答率（正答数 / 出題リストの決まり字の数）。近接正解は 0.5 点として加算され、正答数は小数になりうる
    - 確認モード全体にかかった時間（1問ごとの回答時間は記録しない）
    - 暗記の枚数設定・制限時間などのセッション情報
    - 直近5回分を LocalStorage に保持し、トップ画面に表示

## UI/UX 要件

- 操作性：直感的なインターフェース
- 視認性：実際の畳の配置に近いビジュアル（陣間・札間隔は相対レイアウトで近似）
- 定位置編集画面：決まり字は縦書き表示（必須）。長い決まり字（5・6文字）はコンパクト表記（例：`よのなかよ` → `よの・よ`）
- **スマホでの拡大**：盤面は画面幅に合わせて自動縮小（`--fuda-w`）。より大きく見る必要があるときは **ブラウザのピンチイン/ピンチアウト** に任せる（`viewport` で拡大を禁止していない）。アプリ内の専用陣ズーム UI は設けない

## 非機能要件

- **デプロイ先**：GitHub Pages（サーバー・DB なし）
- **公開 URL**：`https://kg9n3n8y.github.io/anki_practice/`（`VITE_BASE_PATH=/anki_practice/`、GitHub Actions で `dist/` をデプロイ）
- **データポータビリティ**：自陣定位置のコンパクトテキスト取りこみ（トップ）・コピー（定位置編集画面）
- **オフライン**：Service Worker + Web App Manifest（表示名「かるた暗記練」。アイコン `public/pwa-icon-180.png` 等）
- **パフォーマンス**：札の表示・画面遷移がスムーズであること

## 技術スタック

実装時の選定（ドキュメント上の目安）：

- フロントエンド：Vite + React（静的ビルド、GitHub Pages 向け）
- ドラッグ＆ドロップ：dnd-kit（定位置編集画面）
- ストレージ：ブラウザ LocalStorage
- 詳細（JSON スキーマ等）：`4.技術・データ仕様.md` を参照
